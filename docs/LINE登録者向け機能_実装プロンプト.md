# LINE登録者向け追加機能の実装プロンプト

## 概要

診断結果画面で「LINEに保存する」を押したユーザーが、LINE友だち登録と同時に次の情報をすぐ受け取れるようにする機能を実装するためのプロンプトです。

---

## 前提（フロント側で渡すデータ）

- **`top10`**: 診断結果の**上位10駅**（駅名の配列）。表示用。例: `['目黒','自由が丘','大岡山', ...]`
- **`current`**: ユーザーが入力した「現在の最寄り駅」（駅名・文字列）
- **`currentRank`**: 現在の最寄り駅の順位。**50位までを確認**し、1〜50の数字、または50位に含まれない場合は **`out`（圏外）**

「LINEに保存する」クリック時、次のいずれかの形でバックエンドへ渡す想定です。

- **A) クエリ付きランディングURL**:  
  `GET /line-register?top10=駅1,駅2,...,駅10&current=現在の最寄り駅&currentRank=5`（または `currentRank=out`）  
  このURLを開いたあと、サーバーでパラメータを保存し、LINE友だち追加URLへリダイレクトする。
- **B) API + コード方式**:  
  `POST /api/line-register` に `{ top10: string[], current: string, currentRank: string }` を送り、サーバーが一時コードを返す。

---

## 実装してほしいこと

### 1. データの受け取り・保存

- 上記 A または B の方法で、**診断結果10位までの駅一覧（`top10Stations`）** と **現在の最寄り駅（`current`）** を受け取る。
- 受け取ったデータを、LINEの「友だち追加」イベントと紐づけられる形で保存する。
  - **A の場合**: ランディングアクセス時にセッションや一時IDを発行し、そのIDに `top10` と `current` を紐づけて保存。その後、LINE公式アカウントの友だち追加URLへリダイレクト。
  - **B の場合**: 発行した一時コードをキーに、`top10` と `current` を保存（有効期限付きが望ましい）。

### 2. 「今住んでいる駅」の順位（フロントで算出済み）

- フロントでは**診断結果50位まで**の駅リストを元に順位を算出している。
- **`currentRank`** をそのまま利用する。
  - **1〜50**: 現在の最寄り駅が診断結果の何位かを表す数字。
  - **`out`**: 50位までに含まれていない場合＝**「圏外」**として表示する。

### 3. LINE登録と同時の送付

- LINEの **友だち追加（follow）イベント** または **登録直後にユーザーが1通目を送ったタイミング** で、次の内容をそのユーザーに送る。
  1. **診断結果10位までの駅リスト**（1位〜10位の駅名）
  2. **「今住んでいる駅」の順位**  
     例: 「あなたの現在の最寄り駅（○○駅）は、今回の診断では○位でした。」

- 送付は、LINE Messaging API の「プッシュメッセージ」または「応答メッセージ」で実装する。

### 4. 紐づけの方法（参考）

- **A ランディングURL方式**:  
  友だち追加時点では「どのセッションのユーザーか」が分からないため、  
  - ランディングで「このブラウザ用の一時トークン」を発行し、Cookie やローカルストレージに保存したうえで LINE へリダイレクト、  
  - その後、LINE内で「診断結果を送る」などのメッセージやリッチメニューから「送って」を押してもらい、そのタイミングで同じブラウザからトークン付きでバックエンドを呼ぶ、  
  といった「ブラウザとLINEの両方を使う」フローが必要になる場合がある。
- **B コード方式**:  
  ユーザーがLINEで「受け取った6桁コード」を送信 → バックエンドでコードに紐づく `top10` と `current` を取得 → 順位を計算してメッセージを返す、という流れにすると実装しやすい。

## LINEメッセージで使用する定型フォーム

送付するLINEメッセージは、次の構成で統一する。

| ブロック | 内容 |
|----------|------|
| **ヘッダー** | 居住地最適化アルゴリズム 診断結果 |
| **メイン** | 現在の【現在の駅名】は、23区中 第【順位】位です。 |
| **サブ** | あなたに最適な駅 TOP3：1.【1位の駅名】, 2.【2位の駅名】, 3.【3位の駅名】 |
| **フッター** | ボタン「詳細レポートとコンシェルジュ相談を開始」（URLリンク） |

### 差し込みルール

- **【現在の駅名】** → `current`（ユーザーが入力した「現在の最寄り駅」）
- **【順位】** → `currentRank` が 1〜50 のときは「第○位」、`out` のときは **「圏外」**（「第」「位」は付けない）  
  - 例: 「23区中 第5位です。」／「23区中 圏外です。」
- **【1位の駅名】【2位の駅名】【3位の駅名】** → `top10` の先頭3件（`top10[0]`, `top10[1]`, `top10[2]`）
- **フッターのURL** → 詳細レポート・コンシェルジュ相談用のランディングURL（運用で決定）

### テキスト例

```
居住地最適化アルゴリズム 診断結果

現在の大岡山は、23区中 第3位です。

あなたに最適な駅 TOP3：1.目黒, 2.自由が丘, 3.大岡山

[詳細レポートとコンシェルジュ相談を開始]（ボタン・URL）
```

`currentRank` が `out` のときのメイン例：「現在の○○は、23区中 圏外です。」

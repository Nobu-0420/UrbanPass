# 京急蒲田・新代田を選択したときの所要時間算出の流れ

## 「両方の勤務地へ経路がある候補駅がありません」が出る理由と対処

このメッセージは、**勤務地A・Bのどちらか（または両方）から、候補駅への経路が1件も取れなかった**ときに表示されます。

### 原因のパターン

1. **APIキー未設定で自前グラフを使用している場合**
   - 路線データ（`network_data.json`）には **「蒲田」** はありますが **「京急蒲田」** という名前の駅はありません。
   - そのため、勤務地に「京急蒲田」を選ぶと、グラフ上で経路が計算できず、候補が0件になっていました。
   - **対応**: コード側で **京急蒲田 → 蒲田** のエイリアスを入れ、API未使用時は「京急蒲田」を「蒲田」として経路計算するようにしています。これで API キーがなくても結果が出ます。
   - それでも出ない場合は、ブラウザのコンソール（F12）に **`[経路] 勤務地A「京急蒲田」から経路が取得できません`** のようなログが出ていないか確認してください。

2. **APIキーを設定してAPI経路を使っている場合**
   - すべての候補駅について、Google Directions API で経路を取得します。
   - 発着の表記が API で解釈できず **ZERO_RESULTS** や **NOT_FOUND** になると、その組み合わせは「経路なし」になり、両方から経路が取れた候補が0件になることがあります。
   - **対応**: コンソールに **`[Directions API] 経路未取得`** や **`status: "ZERO_RESULTS"`** が出ていないか確認し、必要に応じて `STATION_ADDRESS_FOR_API` で駅名の正規化を追加してください。

### 確認手順

1. **F12** で開発者ツールを開き、**コンソール**タブを開く。
2. もう一度「計算」を実行する。
3. 次のログを確認する。
   - **`[経路] 勤務地A「…」から経路が取得できません`**  
     → API未使用で、その駅がグラフに無い／エイリアスでも解決していない。
   - **`[Directions API] 経路未取得`** や **`status: "ZERO_RESULTS"`**  
     → APIで経路が返ってきていない。発着の住所・駅名表記を確認。

---

## 前提

- **API キーあり**（`config.json` の `googleMapsApiKey` が設定されている）ときは、経路・所要時間は **Google Directions API（transit）** で取得します。
- **API キーなし**のときは、自前の **Dijkstra（network_data.json の路線グラフ）** で算出します。この場合はグラフに含まれる駅だけが対象です。

## 京急蒲田・新代田を「勤務地」として選んだとき（計算ボタン押下）

### 1. API 利用時（推奨）

1. **候補リスト**  
   候補は **STATION_LIST_FOR_DROPDOWN**（TOP20 ＋ CSV の 23 区駅）です。京急蒲田・新代田はグラフに無くても、この一覧にあれば候補になります。

2. **経路取得**  
   勤務地 A（例: 京急蒲田）・勤務地 B（例: 新代田）について、**各候補駅**への経路を次のように取得します。
   - `getTravelTimeBetweenStations('京急蒲田', 候補駅名)`  
   - `getTravelTimeBetweenStations('新代田', 候補駅名)`  
   内部で **Google Directions API（mode=transit）** を呼び、  
   「京急蒲田駅 東京都」→「○○駅 東京都」のように発着を指定して経路を取得します。

3. **レスポンスの利用**  
   API からは次のような値が返ります。
   - **所要時間（分）**: 乗り換え含む全体の時間
   - **乗り換え回数**: 乗り換え回数
   - **徒歩時間（分）**: 駅〜乗り換え〜駅の徒歩の合計  

   これらをそのまま **Family Index のスコア計算**に使います。  
   - `timePart` = (勤務地A→候補の時間 × weightA + 勤務地B→候補の時間 × weightB) / 5  
   - 時間には API の所要時間＋徒歩時間を使います。  
   - 乗り換えペナルティ = (A→候補の乗り換え回数 + B→候補の乗り換え回数) × 設定値  

4. **京急蒲田・新代田について**  
   - 駅名は「京急蒲田」「新代田」のまま API に渡し、「京急蒲田駅 東京都」などに補正してリクエストします。  
   - 自前グラフに無い駅でも、Google が経路を返せば正しく所要時間・乗り換えが入り、候補としてスコア付けされます。

### 2. API 未使用時（Dijkstra）

- 候補は **STATION_LIST**（network_data.json から作ったグラフ上の駅のみ）です。
- **京急蒲田** や **新代田** がグラフに無い、または経路が繋がっていない場合、  
  それらを勤務地に選んでも「その駅から候補への経路」が無いため、  
  該当する候補はスコア計算から外れ、結果に出ないことがあります。
- 正確に扱いたい場合は、**API キーを設定して API 経路を使用する**運用にしてください。

---

## まとめ

| 項目           | API あり                         | API なし（Dijkstra）              |
|----------------|----------------------------------|-----------------------------------|
| 京急蒲田・新代田を勤務地にしたとき | 経路・所要時間を API で取得し、全候補をスコア計算 | グラフに無い／経路なしだと候補が計算されない |
| 候補の範囲     | STATION_LIST_FOR_DROPDOWN        | STATION_LIST（グラフ上の駅のみ）   |

**京急蒲田や新代田のように自前グラフで扱いづらい駅も、API 利用時に正確な経路・所要時間で計算されます。**

## 羽田空港第１・第２ターミナル・羽田空港（第３ターミナル）で結果が出ない場合

### 原因

- **自前グラフ（API未使用）**: 従来、路線データ（`network_data.json`）に **京急空港線** が含まれておらず、羽田空港関連の駅がグラフ上に存在しませんでした。そのため、勤務地に「羽田空港第１・第２ターミナル」や「羽田空港」を選ぶと経路が0件になっていました。

### 対応内容

1. **network_data.json に京急空港線を追加**  
   - 蒲田 → 糀谷 → 大鳥居 → 穴守稲荷 → 天空橋 → 羽田空港第３ターミナル → 羽田空港第１・第２ターミナル の順で定義しました。  
   - 蒲田で JR京浜東北線・東急線などと接続するため、羽田空港から23区の他駅への経路が計算されます。

2. **グラフ用エイリアス（GRAPH_STATION_ALIAS）**  
   - 「羽田空港」→ グラフ上の「羽田空港第３ターミナル」  
   - 「羽田空港第1・第2ターミナル」（半角）→「羽田空港第１・第２ターミナル」（グラフ上の正式名）  
   として経路計算するようにしています。

3. **API用の住所補完（STATION_ADDRESS_FOR_API）**  
   - 羽田空港第１・第２ターミナル、羽田空港第３ターミナル、羽田空港 それぞれについて、API検索用の住所文字列を登録しています。

上記により、API未設定時でも「羽田空港第１・第２ターミナル」「羽田空港」を勤務地に選んだ場合に結果が表示されます。

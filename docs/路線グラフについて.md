# 路線グラフとは（23区網羅について）

## 路線グラフとは

**路線グラフ**は、診断で「駅Aから駅Bまで何分・何回乗り換えか」を計算するために使う**データ**です。

- **中身**: 「どの路線に、どの駅がどの順で並んでいるか」を路線ごとに定義したもの。
- **index.html 内の定義**:
  - **LINE_STATIONS**: 路線ID → 駅IDの配列（隣り合う駅の並び）
  - **STATION_DEFS**: 駅ID → 駅名（漢字）・区
- **使い方**: このデータから「隣接グラフ」を組み、ダイクストラ法で経路・所要時間・乗り換え回数を算出。**このグラフに含まれる駅だけ**が「勤務地」として選べ、かつ「候補駅」として表示される。

つまり、**入力で選べる駅**（CSV由来）と、**経路計算の対象になる駅**（路線グラフ由来）は別で、**グラフにない駅を勤務地にすると「候補駅がない」**になります。

---

## 23区内の全路線を網羅すべきか

**はい。**  
「23区の全駅から勤務地を選んで診断する」のであれば、**23区内を通る路線は原則すべて路線グラフに含める**必要があります。

- 網羅していないと、グラフにない駅を選んだときに「候補駅がありません」になる（例: 以前の新代田・蓮沼）。
- 逆に、23区の全路線・全駅がグラフに入っていれば、入力一覧（CSV）にあるどの駅を選んでも経路が計算され、候補が表示されます。

---

## 現在の状態（網羅していない）

現在の **LINE_STATIONS** は**一部の路線だけ**を手動で定義したものです。

**含まれている主な路線の例**  
山手線、中央線、銀座線、丸ノ内線、東西線、千代田線、南北線、半蔵門線、日比谷線、有楽町線、東急大井町線・目黒線・田園都市線、京王井の頭線、東急多摩川線、JR京浜東北線（品川〜蒲田） など。

**含まれていない路線の例**  
- JR: 埼京線、湘南新宿ライン、常磐線、総武線、京葉線 など
- 地下鉄: 副都心線、大江戸線 など
- 私鉄: 西武線、東武線、京成線、都営新宿線・三田線・浅草線・大江戸線、小田急、東急東横線・世田谷線・池上線 など

このため、**23区「すべて」の路線は入っておらず、グラフにない駅を選ぶと候補が出ません。**

## 23区を網羅するには

### 1. データを外から用意する（推奨）

23区の「路線ごとの駅並び」を**外部データ**で持つと運用しやすいです。

- **例**: JSON や CSV で「路線名 / 路線ID」「駅並び（駅名 or 駅ID）」を定義したファイルを用意する。
- **例**: [国土交通省のオープンデータ](https://www.mlit.go.jp/sogoseisaku/transport/sosei_transport_tk_000016.html) や、[駅データ](https://ekiworld.net/) などから「路線・駅順」が分かるデータを加工して使う。
- **実装**: 起動時（またはビルド時）にそのファイルを読み、`LINE_STATIONS` / `STATION_DEFS` を**生成**する。既存の経路探索ロジック（`buildGraph` ～ `dijkstraFrom`）はそのまま使える。

### 2. index.html のグラフを手で拡張する

今のように **LINE_STATIONS** と **STATION_DEFS** を手で増やす方法です。

- 23区を通る路線を1本ずつ調べ、駅の並び順と区間時間を定義する。
- 作業量が多く、抜け・間違いも起きやすいので、**本数が少ない路線や「よく問い合わせの出る駅」から優先**して追加する運用が現実的です。

### 3. 入力側をグラフに合わせる

「経路計算できる駅」だけを入力候補にする方法です。

- 勤務地の入力一覧を、**路線グラフに含まれる駅だけ**に絞る（例: `STATION_LIST` 由来のリストだけ使う）。
- その代わり「23区の全駅」は選べず、「グラフに入っている範囲の駅」に限定されます。

## まとめ

| 項目 | 説明 |
|------|------|
| **路線グラフ** | 路線ごとの「駅の並び」と駅マスタ。経路・所要時間・乗り換え計算の元データ。 |
| **23区を網羅すべきか** | はい。23区の全駅を勤務地候補にするなら、23区内の路線は原則すべてグラフに含める必要がある。 |
| **現状** | 一部路線のみ定義されており、未登録の路線・駅を選ぶと「候補駅がない」になる。 |
| **今後の拡張** | 外部の路線・駅データを読み込んでグラフを組み立てる方式にすると、23区全線を網羅しやすい。 |

必要であれば、**「路線・駅並び」のJSON/CSV仕様**や、**既存の buildGraph / dijkstraFrom に組み込む手順**も別ドキュメントにまとめられます。
